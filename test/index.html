The url needs to have the following parameters: access_token, and descriptor_id. DescriptorId can be specified in the VeLib init aswell

example: localhost:8080/?descriptor_id=some_descr_id&access_token=ey423...

For testing, you can get the access token of a regular account when it sends requests to Verified, while you are using the app. Check the network XHR tab while using the page to see the Authorization header, or get it from local storage. It shouldn't include the JWT part. Just "ey..." .
The descriptor id will be provided as per your business requirements for the specific template.
